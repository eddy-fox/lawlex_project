## 🎨 2. 전체 화면 작업

### 2-1. 디자인 통일성 유지

```
먼저 화면 작업에 대해 말씀드리겠습니다.

저희 팀은 각자 자신이 맡은 기능에 맞춰서 
디자인을 구상했고,

저는 그 디자인들을 바탕으로 
HTML과 CSS로 페이지들을 구현하면서 

각자의 디자인이 페이지가 바뀔 때마다 
느낌이 다르지 않도록 

색상, 폰트 같은 부분을 
하나의 서비스처럼 통일감 있게 보이도록 했습니다.
```

### 2-2. 통일성 유지 작업 상세

```
**구체적인 작업 내용:**

1. **공통 스타일 시스템 구축**
   - 색상: 브랜드 컬러, 텍스트 컬러 등을 CSS 변수로 통일
   - 폰트: Gooorm Sans 폰트를 전체에 적용
   - 간격 및 크기: 일관된 여백과 폰트 크기 적용

2. **레이아웃 통일**
   - 헤더, 푸터 등 공통 컴포넌트 스타일 통일
   - 카드형 레이아웃, 버튼 스타일 등 재사용 가능한 패턴 정립

3. **페이지 간 일관성**
   - 모든 페이지에서 동일한 색상 팔레트 사용
   - 동일한 폰트와 타이포그래피 시스템 적용
   - 사용자가 페이지를 이동해도 
     하나의 통합된 서비스를 이용하는 느낌 제공

이러한 작업을 통해 사용자 경험을 개선하고 
전문적인 서비스로 보이도록 구현했습니다.
```

## 🏆 3. 랭킹 기능

### 3-1. 랭킹 시스템 개요

```
이어서 랭킹입니다.

랭킹에는 변호사들의 활동을 
한눈에 볼 수 있도록 
두 가지 기준으로 랭킹을 나눴습니다.

좋아요를 많이 받은 변호사와 
답변을 많이 작성한 변호사 기준으로 
각 상위 10명까지 보여주도록 했습니다.
```

### 3-2. 랭킹 기능 상세

```
**랭킹 표시 방식:**

1. **필터 기능**
   - 상단에 "좋아요순"과 "답변수순" 버튼 제공
   - 사용자가 원하는 기준을 선택할 수 있음

2. **순위 표시**
   - 각 기준에 따라 상위 10명의 변호사 표시
   - 1위, 2위, 3위는 특별히 강조 표시
   - 순위 번호, 변호사 이름, 좋아요 수 또는 답변 수 표시

3. **변호사 프로필 연결**
   - 변호사를 클릭하면 
     그 변호사의 정보를 볼 수 있는 페이지로 이동하여 볼 수 있습니다
   - 변호사 상세 정보 확인 가능
```

### 3-3. 도움이 많이 된 답변 섹션

```
이어서 랭킹 하단에는 
"도움이 많이 된 답변"을 보여주는 영역도 있습니다.

**주요 기능:**

1. **답변 선별**
   - 상담글 중에서 좋아요를 많이 받은 답변들만 
     따로 모아서 보여줍니다

2. **미리보기 제공**
   - 상담글 제목과 답변 일부를 
     함께 넣어서 어떤 상담에 대한 답변인지 
     대략적인 내용은 어떤지 
     간단한 미리보기 정도는 확인할 수 있도록 하였습니다

3. **상세 내용 확인**
   - 답변 카드를 클릭하면 
     상담글 내용을 볼 수 있도록 이동합니다
   - 답변 전체 내용과 원글을 함께 볼 수 있습니다

4. **추가 정보**
   - 각 답변에는 좋아요 수, 
     답변 작성 변호사 정보, 
     답변 작성일 등이 표시됩니다

이를 통해 사용자들이 
가장 도움이 되는 답변을 쉽게 찾을 수 있도록 했습니다.
```

## 📝 4. 문의 기능

### 4-1. FAQ 페이지

```
이어서 문의입니다.

문의 메뉴에 들어가면 가장 먼저 
FAQ 화면이 보이며

"문의글 보기" 버튼을 눌러 
이용자들의 문의를 보는 목록 페이지로 이동하게 됩니다.
```

**FAQ 주요 내용:**
- 회원가입 방법
- 상담 글 수정/삭제 가능 여부
- 포인트 충전과 사용 방법
- 채팅 상담 진행 방식
- 변호사 회원 등록 방법
- 문의글 확인 방법
- 관리자 문의 방법
- 변호사 답변 작성 방법
- 회원 탈퇴 방법

**FAQ 기능:**
- 아코디언 형식으로 질문 클릭 시 답변 펼침
- Q&A 형식으로 구성

### 4-2. 문의글 목록 페이지

```
**문의글 목록 페이지 특징:**

1. **문의글 확인**
   - 이곳에서는 다른 이용자들이 
     남긴 서비스에 대한 문의들을 확인할 수 있습니다

2. **비밀글 기능**
   - 비밀글은 본인 이외에는 볼 수 없습니다
   - 비밀글 표시: 🔒 아이콘과 함께 표시
   - 작성자 본인과 관리자만 확인 가능

3. **답변 상태 확인**
   - 답변이 되었는지 확인하실 수 있습니다
   - 답변 완료: "✓" 표시
   - 답변 대기: "-" 표시

4. **내 문의글 모아보기**
   - 자신이 적은 문의 글들을 모아 볼 수 있도록 하였습니다
   - "내가 작성한 문의 보기" 버튼 제공
   - 로그인한 사용자 본인이 작성한 문의글만 필터링

5. **검색 기능**
   - 문의글 제목 및 내용으로 검색 가능
```

### 4-3. 문의글 작성

```
**문의글 작성 기능:**

1. **접근 권한**
   - 문의 작성은 로그인 이용자만 이용 가능하도록 해두었습니다
   - 비로그인 상태에서 글쓰기 시도 시 로그인 페이지로 이동

2. **작성 폼 구성**
   - 제목 입력
   - 내용 입력 (텍스트 영역)
   - 비밀글로 작성 옵션 (체크박스)
   - 작성/취소 버튼

3. **비밀글 옵션**
   - 비밀글로 작성 시 본인과 관리자만 볼 수 있음
```

### 4-4. 문의글 상세 및 답변

```
**문의글 상세 페이지:**

1. **문의 내용 확인**
   - 제목, 작성일, 공개글/비밀글 태그 표시
   - 문의 내용 전체 확인

2. **답변 확인**
   - 답변 완료 시 답변 내용과 답변 날짜 표시
   - 답변 대기 중일 경우 안내 메시지 표시

3. **권한 관리**
   - 작성자는 자신의 문의글 수정/삭제 가능
   - 관리자는 모든 문의글 확인 및 답변 작성 가능
   - 비밀글은 작성자와 관리자만 확인 가능

4. **답변 작성 (관리자)**
   - 관리자는 문의글 상세 페이지에서 직접 답변 작성
   - 답변 작성 후 자동으로 "답변완료" 상태로 변경
```

### 예상 질문 및 답변

**Q: 디자인 통일성을 위해 구체적으로 어떤 작업을 했나요?**
A: CSS 변수를 사용하여 색상과 폰트를 통일하고, 공통 컴포넌트 스타일을 만들어 모든 페이지에 적용했습니다. 특히 색상 팔레트와 폰트 시스템을 일관되게 적용하여 하나의 서비스처럼 보이도록 했습니다.

**Q: 랭킹은 실시간으로 업데이트되나요?**
A: 네, 변호사가 좋아요를 받거나 답변을 작성할 때마다 데이터베이스에 반영되며, 랭킹 페이지를 새로고침하면 최신 순위가 반영됩니다.

**Q: 비밀글이 정말 다른 사람에게 보이지 않나요?**
A: 네, 비밀글은 데이터베이스에서 작성자 ID와 관리자 권한을 확인하여, 본인과 관리자만 조회할 수 있도록 백엔드에서 권한 체크를 하고 있습니다.

**Q: 도움이 많이 된 답변은 어떻게 선별하나요?**
A: 모든 답변 중에서 좋아요 수를 집계하여 상위 10개를 선별하며, 활성 변호사와 활성 게시글만 대상으로 합니다.
